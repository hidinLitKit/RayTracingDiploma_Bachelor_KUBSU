# Трассировка лучей в Unity URP

Подсистема эффектов реального времени на аппаратной трассировке лучей (DXR) для Unity 6 / URP. Гибридный подход: базовый кадр строит растеризация, а тени, отражения и объёмный свет считаются лучами.

## Эффекты

- **Мягкие тени (PCSS)** — ширина полутени оценивается трассировкой по расстоянию до перекрывающего объекта, фильтр с переменным радиусом.
- **Отражения** — рекурсивные, сила по Френелю (Шлик), конус лучей для шероховатых поверхностей, отдельная обработка воды.
- **Объёмный свет (god rays)** — маршевый проход с фазовой функцией Хеньи–Гринштейна, видимость источника через теневой луч.

## Идея

Эффекты считаются в пониженном разрешении и с малым числом лучей, поэтому по отдельности шумят и мерцают. Устойчивость в реальном времени даёт общий слой денойза: билатеральная фильтрация, временное накопление с репроекцией и повышение разрешения с учётом глубины. Денойз вынесен в ядро и переиспользуется всеми эффектами.

## Архитектура

- `RaytracePassBase` / `RaytraceFeatureBase` — общее ядро проходов и Scriptable Renderer Feature.
- `RaytraceDataManager` — построение и хранение `RayTracingAccelerationStructure`.
- `RaytracePreparePass` — подготовка матриц камеры и общих параметров кадра.
- `RaytraceConfig` (ScriptableObject) — ссылки на шейдеры и компьюты.
- Эффекты подключаются как отдельные Renderer Features.


## Эдиторские тулзы

- **Texture Repacker** — распаковка масок сторонних ассетов (ORM/RMA/…) в формат нашего Lit.
- **Terrain To Mesh** — конвертация Terrain в статический меш для участия в трассировке.
- **RTAS Audit** — аудит сцены: какая геометрия раздувает ускоряющую структуру.

## Требования

- Unity 6 (URP, RenderGraph).
- GPU с аппаратной трассировкой лучей и D3D12 (Windows).
- При отсутствии поддержки RT проходы не исполняются.
